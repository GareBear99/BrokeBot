25+ Coin Funding‑Event Engine (No API Keys Required)

Phase‑1 mirrors Doman.ai’s “many small, net‑positive executions” — but for funding‑rate capture across 25+ coins, using wallet-based access and public on-chain/read endpoints.

Minimum bucket: $5+ Leverage: 2×–5× isolated Scaling: more coins + more events Goal: double→triple monthly (aspirational) Date: 2025-12-30

What “Phase‑1” means (plain English)

  • Only take trades that are net‑positive after all costs. If fees/spread/slippage erase the edge, skip.
  • Win by repetition, not prediction. Lots of tiny wins across many instruments.
  • Keep risk survivable. Small buckets, isolated margin, hard daily stop rules.
Important: “Double→triple monthly” is not guaranteed. It requires frequent funding extremes, low execution costs, and strict risk caps so a few spikes don’t erase weeks of capture.

Edge Type

Broke Bot Phase‑1 is a funding-event engine: open a delta‑neutral position near funding time only when the capturable edge is large enough to beat all costs.

Net per event ≈ Notional × |FundingRateCaptured| − (Fees + Spread + Slippage + SafetyBuffer)

Scaling happens by increasing qualified events (more coins × more platforms), not by pushing leverage.

No API keys: multi‑platform data + wallet execution

Public read sources (no keys)

  • On‑chain contracts: read funding, OI, and prices via public RPC.
  • Protocol public “info” endpoints: often keyless for reads but rate-limited.
  • Indexer endpoints: some ecosystems publish read-only HTTP endpoints.

Execution (no exchange API keys)

  • Trades signed by a user wallet (MetaMask / WalletConnect style flows).
  • Bot submits transactions through RPC (paid RPC recommended later for stability).
  • TRON rail: USDT‑TRC20 transfers can move capital cheaply between venues.
Reality check: “No API keys” ≠ “no limits.” Public endpoints can rate-limit, RPC can fail, and some venues require accounts/KYC. Plan for caching + fallback providers.

Qualification gates (prove it works first)

Gate 1 — Paper Mode (minimum 24 hours)

  • Run event selection without placing trades for 24h.
  • Must record at least 1 positive net event with realistic fee/spread estimation.

Gate 2 — Small Live Mode

  • Start with Bucket A ($5) across ~5 coins on 1 platform.
  • Daily loss caps: stop after 2 failed exits or after -10% of Phase‑1 bankroll.

Gate 3 — Expansion to 25+ coins & multi‑platform

  • Add coins only if they meet liquidity/spread/volatility limits.
  • Expand platforms only when monitoring + execution is stable.

How to chase “double→triple monthly” (aspirational plan)

Three levers — in this order:

  1. Increase qualified event count (25+ coins × multiple platforms).
  2. Increase capture quality (filters + execution → lower slippage and fewer spike losses).
  3. Increase bucket count (parallelism) — not leverage.

Phase‑1 signal selection

  • Funding extreme above threshold AND stable (not whipping violently).
  • Spread below max, adequate depth, and volatility compression into funding time.
  • Arm intent 10–90 minutes before funding; cancel if any filter fails.
  • Enter delta‑neutral hedge; hold through snapshot; exit immediately after.

Risk controls (survivability)

  • Isolated margin only.
  • Per-coin max exposure (e.g., 5% of Phase‑1 bankroll).
  • Daily stops: stop after 2 bad exits or -10% bankroll on the day.
  • Kill switch if RPC/quotes stale.
  • Emergency unwind on volatility spikes; log incident.
